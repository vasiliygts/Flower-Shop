# FlowerShopApp — інформаційна система «Магазин квітів»

Настільний WPF-додаток для автоматизації роботи магазину квітів: облік товарів, клієнтів, замовлень, поставок, залишків та формування базових звітів на основі СУБД PostgreSQL.

Проєкт розроблено як  робота з напрямку «Бази даних» і демонструє повний цикл створення прикладної інформаційної системи: від проєктування моделі БД до клієнтського застосунку для щоденної роботи менеджера магазину.

---

## Основний функціонал

- **Головне вікно**
  - Меню навігації по основних розділах: Замовлення, Товари, Клієнти, Постачальники, Закупки, Звіти.
  - Статус-бар зі станом підключення до БД PostgreSQL (успішне підключення / помилка).

- **Товари**
  - Перегляд списку доступних товарів (квіти, букети, аксесуари тощо).
  - Додавання, редагування та видалення товарів (CRUD).
  - Зберігання інформації: назва, тип, опис, ціна, термін придатності.

- **Клієнти**
  - Облік клієнтів магазину з контактними даними (ПІБ, телефон, email, адреса).
  - Створення, редагування та видалення клієнтів.
  - Подальше використання клієнта при оформленні замовлення.

- **Замовлення**
  - Створення нового замовлення з прив’язкою до клієнта.
  - Додавання кількох товарних позицій (деталі замовлення: товар, кількість, ціна на момент замовлення, закупівельна ціна).
  - Редагування замовлення та його деталей.
  - Видалення замовлення разом із пов’язаними деталями (з використанням транзакції).
  - Зберігання статусу, способу оплати та адреси доставки.

- **Постачальники та поставки**
  - Облік постачальників (контрагентів, що постачають товар).
  - Реєстрація поставок від постачальників.
  - Зберігання номенклатури поставки (деталі поставки: товар, кількість, закупівельна ціна).
  - Оновлення складських залишків на основі поставок і продажів.

- **Звіти та аналітика**
  - Формування звіту з продажів за вибраний період:
    - клієнт, товар, кількість, закупівельна та продажна ціна;
    - можливість подальшого розрахунку прибутку.
  - Звіт за популярними товарами (сума проданих одиниць за період).
  - Звіт за активними клієнтами (кількість замовлень, кількість куплених товарів).
  - Підготовка бази для розширення аналітики (залишки, поставки, активність постачальників).

## Технології

- **Мова програмування:** C#
- **UI-фреймворк:** WPF (Windows Presentation Foundation)
- **Платформа:** .NET (настільний застосунок)
- **СУБД:** PostgreSQL
- **Доступ до БД:** Npgsql (ADO.NET провайдер для PostgreSQL)
- **Архітектура застосунку:**
  - шаблон «Repository» для роботи з таблицями БД;
  - окремі моделі доменних сутностей (товари, клієнти, замовлення, деталі, постачання, звіти);
  - окремі вікна / форми для кожного розділу (Products, Customers, Orders, Suppliers, Supplies, Reports).

## Структура проєкту

(Назви папок можуть трохи відрізнятися, підкоригуйте під свій репозиторій.)

- `FlowerShopApp/`
  - `App.xaml`, `MainWindow.xaml` — точка входу, головне вікно з меню та статус-баром.
  - `Data/` — репозиторії роботи з БД:
    - `ProductRepository` — товари;
    - `CustomerRepository` — клієнти;
    - `OrderRepository`, `OrderDetailRepository` — замовлення і їх деталі;
    - `ReportRepository` — побудова звітів (продажі, популярні товари, активні клієнти);
    - `DbConnectionHelper` — допоміжний клас для створення підключення до PostgreSQL.
  - `Models/` — моделі сутностей (Product, Customer, Order, OrderDetail, SalesReportItem, PopularProductItem, ActiveCustomerItem тощо).
  - `Forms/` — WPF-вікна для роботи з розділами системи (ProductsWindow, CustomersWindow, OrdersWindow, SuppliersWindow, SuppliesWindow, ReportsWindow).
  - `App.config` (або `FlowerShopApp.exe.config`) — рядок підключення `PostgresConnection` до БД PostgreSQL.
- `sql/` (за наявності) — SQL-скрипти для створення структури БД та наповнення тестовими даними.

## Модель даних (коротко)

Система реалізує класичну модель інтернет-/офлайн-магазину:

- **Клієнти (Customers)** — покупці з контактними даними.
- **Товари (Products)** — квіти, букети, аксесуари з ціною та описом.
- **Замовлення (Orders)** — оформлені покупки клієнтів.
- **Деталі замовлення (OrderDetails)** — окремі позиції товарів у замовленні.
- **Постачальники (Suppliers)** — постачальники товарів.
- **Поставки (Supplies)** та **деталі поставок (SupplyDetails)** — поповнення складу.
- **Склад / залишки (Inventory)** — поточні залишки товарів.
- **Доставка, працівники (Delivery, Employees)** — можуть бути використані для розширення функціоналу доставки.

### 1. Необхідні інструменти

- Windows 10 / 11.
- Встановлений **.NET SDK** (6.0+ або відповідна версія, під яку зібраний проєкт).
- **Visual Studio 2022** (або інша IDE з підтримкою WPF-проєктів .NET).
